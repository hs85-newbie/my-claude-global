## 실행 방식 선택 기준

### 멀티에이전트 (Agent 도구 사용)
- 독립적 서브태스크 2개 이상 + 병렬 실행 가능한 경우
- 탐색/수집처럼 컨텍스트 오염 없는 작업
- worktree 격리가 필요한 브랜치별 병렬 작업

### 하네스 (직접 실행)
- 이전 단계 결과가 다음 단계에 필요한 순차 의존성이 있을 때
- 동일 파일/컨텍스트를 연속으로 수정하는 작업
- 중간에 사용자 확인이 필요한 단계가 있을 때

### Paperclip으로 위임
- 24시간 이상 걸리는 장기 자율 실행
- 사람의 승인 게이트가 필요한 작업
- 예산/토큰 관리가 필요한 대규모 자동화

## 커밋 규칙
- bisect commit: 논리적 단위 1개 = 커밋 1개
- 타입: feat / fix / chore / refactor / docs
- 조건: 0 TypeScript errors, 0 ESLint errors
- 커밋 메시지 언어: 한국어

---

## 코딩 핵심 원칙
- 투기적 추상화 금지 — 지금 필요한 것만 구현
- 검증은 시스템 경계(사용자 입력, 외부 API)에서만
- pnpm 기본, TypeScript 우선, any 타입 금지
- DRY: 동일 로직 2회 이상 등장 시 즉시 함수 추출
- 함수 최대 30줄 / 파일 최대 300줄 (초과 시 분리)
- 구현 전 검증 기준(테스트 또는 성공 조건) 먼저 정의
- 새 작업 시작 전 무관한 이전 컨텍스트는 /clear로 리셋

## 테스트 원칙
- 새 기능: 단위 테스트 필수 / 외부 연동: 통합 테스트 필수
- 커버리지 80% 미만 PR 머지 금지
- 테스트 없는 코드 = 완성되지 않은 코드

## 에러 처리 원칙
- 외부 API 호출: try/catch + 재시도 최대 3회 (지수 백오프)
- 빈 catch 블록 금지 — 반드시 로그 또는 상위 전파
- 사용자 노출 메시지: 기술 정보 제거, 친화적 표현만
- 상세 규칙: @docs/error-handling.md

## AI 에이전트 자율성 경계

### 확인 없이 자율 실행 가능
- 파일 읽기, 코드 분석, 탐색
- 로컬 파일 생성/수정 (테스트, 문서)
- 테스트 실행, 린트 실행
- 멀티에이전트 서브태스크 생성

### 반드시 사람이 확인 후 실행
- git push, PR 생성
- 외부 API 호출 (데이터 전송)
- 패키지 추가/삭제
- DB 마이그레이션 실행
- 배포 명령 실행
- Paperclip 예산 임계값(1회 $10) 초과 작업

## 규칙 추가 감지 원칙

아래 상황을 감지하면 작업 전에 먼저 보고한다:
- 같은 유형의 실수/버그가 2회 이상 반복될 때
- 현재 규칙으로 판단이 불가능한 상황일 때
- 문서에 없는 새 패턴을 만들어야 할 때
