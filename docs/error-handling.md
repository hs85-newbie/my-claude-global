# 에러 처리 표준

## 외부 API 호출
- try/catch 필수, 빈 catch 금지 (최소 로그 기록)
- 재시도: 최대 3회, 지수 백오프 (1s → 2s → 4s)
- 타임아웃: 기본 5초, 파일 업로드 30초

## 에러 분류
- 4xx: 클라이언트 에러 → 재시도 금지, 사용자에게 원인 안내
- 5xx: 서버 에러 → 재시도 가능, 로그 남기고 fallback
- Network: 재시도 가능, 오프라인 상태 감지 후 UX 처리

## 사용자 노출 메시지
- 기술 스택 정보 노출 금지 (스택 트레이스, DB 에러 등)
- 친화적 한국어 메시지 + 다음 행동 안내 포함
- 예: "잠시 후 다시 시도해주세요. 문제가 계속되면 고객센터로 연락해주세요."

## React 에러 경계
- 페이지 단위 ErrorBoundary 필수 래핑
- fallback UI: 새로고침 버튼 + 홈으로 이동 링크

## 로깅 표준
- 구조화 JSON 로그: { level, message, context, timestamp, traceId }
- ERROR: 즉시 알림 필요 / WARN: 모니터링 / INFO: 주요 흐름 / DEBUG: 개발 전용
