# 테스트 전략

## 테스트 피라미드
- 단위(Unit): 70% — 순수 함수, 유틸리티, 비즈니스 로직
- 통합(Integration): 20% — API 엔드포인트, DB 연동, 외부 서비스
- E2E: 10% — 핵심 사용자 시나리오 (로그인, 결제 등)

## 커버리지 기준
- 전체: 80% 이상 (미달 시 PR 머지 차단)
- 핵심 비즈니스 로직: 95% 이상
- UI 컴포넌트: 60% 이상

## 도구 표준
- 단위/통합: Vitest (pnpm 프로젝트 기준)
- E2E: Playwright
- 커버리지: c8 (Vitest 내장)

## 작성 원칙
- AAA 패턴: Arrange(준비) → Act(실행) → Assert(검증)
- 테스트명: "~일 때 ~해야 한다" 형식
- 외부 의존성: 반드시 Mock/Stub 처리
- 테스트 간 상태 공유 금지 (독립성 보장)

## AI 생성 테스트 검수 기준
- Happy path만 테스트하는 코드 거부
- 엣지케이스 (빈값, null, 최대값) 반드시 포함
- "테스트를 위한 테스트" (로직 없는 통과 테스트) 금지
